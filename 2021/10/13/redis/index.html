Redis 操作

3、编写配置类配置RedisTemplate，指定key和value的序列化对象，不配置可能出现如下问题：

<1>key和value在redisclient中显示有问题，但不影响正常使用，即保存和获取都没有问题。

<2>不能自动保存对象，保存时需要手动将对象转换成json字符串，获取时需要手动将json字符串转换成对象。

<3>具体配置参考：RedisConfiguration.java

基于RedisTemplate操作redis：RedisTemplate由springboot自动创建，直接注入即可。

1、redisTemplate.opsForValue():获取RedisTemplate中操作字符串的对象。

数据结构：

键:值

<1>set(key,value):添加或者修改指定键的字符串，如果键不存在为添加，存在为修改。

<2>set(key, value, timeout, TimeUnit);添加或者修改指定键的字符串，并指定有效时间，如果键不存在为添加，存在为修改。

  1. get(key)：获取指定键的值。
  2. append(key,value):在指定键的内容后面追加指定内容。
  3. increment(key):给指定键的值加1，如果键不存在则会自动创建，且默认值为0。
  4. increment(key,num):给指定键的值加num，num可以为整数和小数。
  5. decrement(key):给指定键的值减1。
  6. decrement(key,num)：给指定键的值减num，num只能为整数。
  7. multiSet(Map map)：将map中的键值对分别保存在redis中。
  8. multiGet(Collection kes)：获取多个key的值。

【注意】可以将对象直接保存在redis中，RedisTemplate的配置类中指定了value的序列化对象为jackson，在保存对象时jackson会自动将对象转换成json字符串，获取值时，jackson会将json字符串自动转换成对象。

2、redisTemplate.opsForList():获取RedisTemplate中操作list列表的对象****。

数据结构：

键：[值1,值2……]

<1>rightPush(key, value):给指定键对应的List列表的右边添加一个值。还有left

<2>rightPushAll(key, String...value):

给指定键对应的List列表的右边添加多个值,还有left。

<3>rightPushAll(key, Collection value)：

给指定键对应的List列表的右边添加多个值，还有left。

<4>rightPop(key):非阻塞模式(有值就获取了,没值就返回null,不等待)获取指定键对应list列表中右边的第一个值，值获取后就会被移除，当list中的所有值都被移除后，list对应的键也会被移除，获取的值为null。还有left。

<5>rightPop(key,时间,单位)：在指定的时间内以阻塞模式获取指定键对应list列表中右边的第一个值，值获取后就会被移除，当list中的所有值都被移除后，list对应的键也会被移除。如果获取不到值，则在指定的时间内阻塞等待，如果获取到了值，则立即解除等待，直接返回。如果指定时间为0，则表示一直等待，直到有数据才解除等待返回数据。

<6>set(key, 下标, 值):设置指定键对应list列表中指定下标的值。

<7>remove(key, 个数, 值):移除指定键对应list列表中指定个数的指定值。

如：remove("list",2,"张三"):移除list这个键对应list列表中2个张三。

<8>size(key):获取指定键的list列表的大小。

<9>indexOf(key,value)：获取指定key中指定value的下标，如果不存在返回null。

还有lastIndexOf。

注意：由于4和5，因此list列表适合保存消息列队中的消息。

3、redisTemplate.opsForSet():获取RedisTemplate中操作set集合的对象。

数据结构：

键：[值1,值2……]

  1. add(key,Object...values):给指定键添加多个值。
  2. members(key):获取指定键的所有值，返回Set集合。
  3. pop(key)：随机获取指定key中的一个值，返回Object，值获取后就会被移除。
  4. remove(key,Object...value):移除指定键对应set集合中的指定值。
  5. difference(key1,key2)：获取两个key不同的值，返回Set集合。
  6. size(key):获取指定键对应set集合的大小。

4、redisTemplate.opsForHash():获取RedisTemplate中操作hash列表的对象****。

数据结构：

键：{键1:值1,键2:值2……}

  1. put(key, hashKey, value);添加或者修改指定键的键值对。
  2. putAll(key,map):添加或者修改指定键的hash列表。如果键不存在则为添加，存在为修改。
  3. get(key, Object hashKey):获取指定键对应hash列表中指定键的值
  4. entries(key):通过键获取hash列表，返回Map。
  5. delete(key,Object...hashKeys):删除指定键对应hash列表中指定键值对。
  6. hasKey(key,hashkey)：判断key对应的map中是否包含hashkey这个键，包含返回true，否则返回false。

5、redisTemplate.opsForZset():获取RedisTemplate中操作zset的对象。

数据结构：

键 值1 分值1(数量)

键 值2 分值2(数量)

键 值3 分值3(数量)

<1>add(key, value, 分值)：添加或修改一个键值对并给该键值对绑定指定分值，根据分值排序

<2>remove(key,Object...value):移除指定键对应的sorted set中的指定值。

<3>removeRange(key,start,end):移除指定键对应的sorted set中指定下标范围的值。

<4>removeRangeByScore(key,min,max):移除指定键对应的sorted set中指定分值范围的值。

<5>range(key,start,end):获取指定键按照分值从小到大指定下标范围的值，返回值的set集合。包括开始，包括结束。

<6>reverseRange(key,start,end):获取指定键按照分值从大到小指定下标范围的值，返回值的set集合。包括开始，包括结束。

<7>incrementScore(key, value, 分值):给指定键中的指定value添加分值。

<8>score(key, value):获取指定键指定值的分值。

<9>rank(key, value);获取指定值在指定键对应的sorted set中按照分值从小到大的下标或者排名，从0开始。返回long类型的下标或排名。

<10>reverseRank(key, value);获取指定值在指定键对应的sorted set中按照分值从大到小的下标或者排名，从0开始。返回long类型的下标或排名。

<11>size(key):获取指定键对应的sorted set的大小。

6、redisTemplate有一些直接操作键的重要方法：

  1. delete(key):删除指定键。
  2. hasKey(key):判断是否有某个键，如果有返回true，否则返回false。
  3. expire(key, timeout, TimUnit):设置键的有效时间。
  4. expireAt(key,Date)：设置键在指定时间过期。
  5. persist(key):设置某个键永久保存，为默认。
  6. keys("通配符")：根据通配符获取指定键。*表示所有。
  7. redisTemplate.keys("*"); 获取所有键
  8. redisTemplate.keys("u*"); 获取所有以u开头的键

springboot整合redis做缓存

1、在启动类或者redis配置类上使用@EnableCaching开启缓存。

注意：@EnableCaching不仅仅可以开启redis缓存，其他缓存也由该注解开启。

2、使用缓存：

1@Cacheable：标注在方法上说明该方法支持数据缓存，在执行方法体之前，先判断key在缓存中是否存在，如果存在则直接返回对应的缓存；如果不存在则执行方法体获取数据，并将方法返回值存储到缓存中。有3个重要属性：

  • value：指定缓存的名字，必须指定，否则报错。
  • key：指定缓存的key，支持SpEl表达式，#root可以省略(建议写，有提示)。
  • condition：指定缓存条件，支SpEl表达式，默认缓存所有。如：condition=”#id<10”

注意：生成key的值=value::key

2@CachePut：标注在方法上说明该方法支持数据缓存，在执行方法体之前，不会检查缓存是否存在，而是每次都会执行方法体，并将方法返回值存储到缓存中。如果Key对应的缓存存在，则更新key对应的value。

用法及其属性同@Cacheable

3@CacheEvict：移除指定缓存中指定key对应的缓存。

用法及其属性同@Cacheable

SpEL表达式：spring表达式语言，通过#取值。

1、methodName：当前方法名，如：#root.methodName

2、args：当前方法参数组成的数组，如：#root.args[0]，获取方法的第一个参数值。

3、caches：当前方法使用的缓存名称数据，如：#root.caches[0]，获取缓存的名称。

4、#stu.sid：获取stu参数中sid属性的值，方法体执行后才获取。

public void test(Student stu){…}

5、#p0.sid：获取第一个参数中sid属性的值，方法体执行后才获取。

public void test(Student stu){…}

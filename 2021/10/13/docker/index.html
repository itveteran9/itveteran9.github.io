目前软件开发的问题(用来引出docker)

1、软件安装问题：开发环境安装一遍所有软件(jdk、mysql、tomcat等)，测试环境也要安装一遍、最后正式环境 (生产环境)还得安装一遍，每换一台机器都得重新安装。

2、环境配置和环境不兼容问题：软件安装后，还得配置每台机器的环境，好不容易环境配好了，发现环境不兼容，在开发环境运行正常，上了测试环境、正式环境却运行不了。

3、提问：要再10台新机器上部署一个springboot项目，该怎么办？

docker 介绍

1、docker是一种虚拟化的容器技术，docker将运行项目的每个部分都当做一个小容器，如：将tomcat当做一个容器，mysql当做一个容器等等。当要使用的时候，就将各个小容器组装起来使用。类似于集装箱一样，将不同的东西放在不同的箱子中装好，最后组装好一次运走。docker就是使用集装箱的概念。(最好带学生看docker的图标，一个乌龟背了很多箱子)

2、docker可以让开发者轻松的将应用以及依赖包打包到一个容器中，然后部署到任何流行的Linux机器上，而不需要关心环境问题。

[3、容器是完全使用沙箱机制]

4、docker容器技术的好处：

启动快

资源占用少

轻量级(体积小)

5、运行镜像：结果会为该镜像创建一个容器实例。

docker run -itd –name 容器名 -p x:y -e 参数 镜像源

案例：运行mysql镜像生成mysql容器：

docker run -itd –name mysql-container -p 3307:3306 -e MYSQL_ROOT_PASSWORD=123 mysql

运行mysql镜像源，生成的容器名为mysql-container，将容器的3306端口映射到宿主机的3307端口，指定root用户的密码为123。

run参数-itd：如果不指定，则启动容器后是一个阻塞窗口。

i：以交互模式运行容器，通常与-t同时使用。

t：为容器重新分配一个伪输入终端，通常与-i同时使用。

d：后台运行容器，并返回容器ID。

如果只显示一个字符串则表示容器创建成功。

docker安装mysql：

1、拉取mysql镜像：

docker pull mysql

2、查看镜像验证是否拉取成功：

docker images

3、运行镜像生成mysql容器：

docker run -itd –name 容器名 -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123 mysql

4、查看容器是否运行成功：

docker ps

进入mysql容器：

docker exec -it mysql容器名 bash

命令登录进入mysql：

mysql -u root -p，回车输入密码。

或者

mysql -uroot -p123，回车。

【注意】ctrl+d表示退出。

use mysql;使用mysql数据库。
